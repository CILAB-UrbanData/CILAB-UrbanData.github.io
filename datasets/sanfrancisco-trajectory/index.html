Dataset Description

The preprocessed bundle contains multiple cities. For this dataset page, we use only sf_data. The trajectories are map-matched onto an OpenStreetMap road network and represented as a sequence of road segments with timestamps.

Files are organized as: map/nodes.shp, map/edges.shp, and trajectory splits (traj_train.csv, traj_valid.csv, traj_test.csv).

Data Schema

Note: The DRPK preprocessed trajectory CSV files are headerless. Interpret fields by column position (1-based).

Trajectory files: traj_train.csv / traj_valid.csv / traj_test.csv

Column Type Meaning Example (truncated)
1 string Moving object id (vehicle/taxi id). edmugrip
2 string Trajectory id. 1212038019:edmugrip
3 string (serialized list) offsets: a serialized list of two tuples (source & destination). Each tuple is (distance_from_seg_start, depart/arrive_time, lon, lat). [(137.9693, 1212038019, -122.39606, 37.792731), (41.6309, 1212038170, -122.40162, 37.793008)]
4 string (serialized list) segment sequence: a serialized list of segment records. Each record is [segment_id, enter_timestamp, avg_speed, duration]. [[6557, 1212037972.113, 2.943, 71.205], [8965, 1212038043.318, 2.943, 15.766], ...]
5 (optional) string (serialized list) Some examples show an extra serialized list of key GPS points like (lon, lat, timestamp, speed). If your files only have 4 columns, this column does not exist. [(-122.39636, 37.79236, 1212038019, 2.942585), ...]

Road network files: map/nodes.shp & map/edges.shp

  • nodes.shp: OSM node information with unique node id.
  • edges.shp: network connectivity information with unique edge id.

Parsing tip: because columns 3–4 contain quoted lists with commas, use a real CSV parser (e.g., Python csv/pandas) instead of splitting by comma.

OSM map data (map/)

  • nodes.shp: OSM node information with unique node id.
  • edges.shp: network connectivity information with unique edge id.
