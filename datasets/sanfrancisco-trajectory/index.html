Dataset Description

The download link contains multiple cities. For this dataset page, we use only sf_data. Trajectories are map-matched to an OpenStreetMap road network and stored as sequences of road segments with timestamps.

File structure (within sf_data): map/nodes.shp, map/edges.shp, and trajectory splits traj_train.csv, traj_valid.csv, traj_test.csv.

Computed statistics (UTC): min 2008-05-17T09:59:09.048Z, max 2008-06-10T09:23:21.931Z. Road segments (edges): 26,659. Total trajectories: 314,507.

Data Schema

Important: traj_*.csv files are headerless and have exactly 4 columns. Interpret fields by column position. Columns 3–4 contain serialized lists (with commas).

Trajectory files: traj_train.csv / traj_valid.csv / traj_test.csv

Column Type Description Example (truncated)
1 string Moving object id (vehicle/taxi identifier). edmugrip
2 string Trajectory id. 1212038019:edmugrip
3 string (serialized list) Offsets: serialized list of two tuples (source & destination). Each tuple: (distance_from_segment_start, depart_or_arrive_time, lon, lat). [(137.9693, 1212038019, -122.39606, 37.792731), (41.6309, 1212038170, -122.40162, 37.793008)]
4 string (serialized list) Segment sequence: serialized list of segment records. Each record: [segment_id, enter_timestamp, average_speed, duration]. The effective segment end time is enter_timestamp + duration. [[6557, 1212037972.113, 2.943, 71.205], [8965, 1212038043.318, 2.943, 15.766], ...]

Road network files: map/nodes.shp & map/edges.shp

  • nodes.shp: OSM node information (graph nodes).
  • edges.shp: OSM edge/segment information (graph edges). Total edges in sf_data: 26,659.

Parsing tip: because columns 3–4 contain list strings with commas, use a proper CSV parser (e.g., Python csv/pandas) rather than splitting by comma.
