Dataset Description

NYC Commuting provides a tract-level view of commuting flows across New York City. Each geographic unit is a census tract, and flows represent the number of workers commuting from a home tract to a work tract. The dataset is derived from the 2015 Longitudinal Employer–Household Dynamics Origin–Destination Employment Statistics (LODES) and NYC PLUTO land-use data, following the setup used in the GMEL model for commuting flow prediction.:contentReference[oaicite:2]{index=2}

For each tract, the dataset aggregates key urban indicators (e.g., building counts, land-use composition, floor area ratios) from PLUTO into tract-level feature vectors, and constructs a geo-adjacency network and distance matrix between tract centroids. These components together support learning geo-contextual embeddings and modeling the spatial interaction between supply (origin) and demand (destination) regions.

This dataset is particularly useful for commuting flow prediction, gravity/deep gravity benchmarking, graph representation learning, and broader urban analytics that require a compact, tract-level OD network of New York City.

Distances (OSRM/OSM): OSRM routing engine based on OpenStreetMap data .

Processed NYC Commuting Dataset

The processed NYC Commuting dataset packages tract-level OD flows into k-means-based tract clusters. For each cluster (on average around 15 tracts), a folder contains four NumPy .npy arrays encoding the OD flow matrix, tract feature matrix, distance matrix, and adjacency matrix. All cluster folders are distributed together as a single compressed archive (~1.02 MB) for direct use in modeling.

Data Schema

The NYC Commuting dataset is stored in a cluster-based layout. All 2,152 census tracts are partitioned into k-means clusters (each cluster containing on average about 15 tracts). For each cluster, there is a folder containing four NumPy .npy arrays. Within a cluster, all arrays share the same tract ordering, so row/column indices are consistent across flows, features, distances, and adjacency.

File Shape Description Example
OD flow matrix (od_flow.npy) (N_c, N_c) Commuting OD flow counts between census tracts inside the cluster. Entry [i, j] is the number of commuters from tract i to tract j in this cluster (rows and columns are indexed by the local tract list of that cluster). e.g., od_flow[3, 7] = 128
Feature matrix (features.npy) (N_c, F) Tract-level feature vectors within the cluster. Each row corresponds to a census tract, and each column corresponds to a feature (e.g., PLUTO-derived building/land-use indicators, density measures, or other socio-economic attributes). e.g., features[3] is the feature vector for tract i = 3 in this cluster
Distance matrix (dist.npy) (N_c, N_c) Pairwise distances between tracts in kilometers (e.g., centroid-to-centroid or OSRM-based travel distance) for tracts within the cluster. Entry [i, j] is the distance from tract i to tract j. e.g., dist[3, 7] = 4.2
Adjacency matrix (adj.npy) (N_c, N_c) Binary or weighted adjacency between tracts within the cluster, representing the underlying graph structure (e.g., geographic contiguity or network connectivity). Entry [i, j] is non-zero if tracts i and j are connected. e.g., adj[3, 7] = 1 if tracts 3 and 7 are neighbors

Conceptually, these cluster-level matrices encode the same information as a global tract-level OD table, feature table, distance matrix, and adjacency matrix, but are partitioned by k-means clusters to create smaller, denser subgraphs that are convenient for training neural network models.
