Dataset Description

This dataset contains origin–destination (OD) mobility flows within New York State, derived from the COVID19USFlows dataset (MIT License) and reformatted into a standardized OD schema compatible with LargeCommutingOD-style experiments. The underlying flows capture movements between census tracts during the COVID-19 period, as described in the associated data descriptor and widely used in mobility and epidemic modeling studies.

In addition to OD flows, each tract is enriched with geographic features derived from OpenStreetMap, such as land-use areas, road network length, and facilities related to transport, food, health, education, and retail. These OSM-based features are aggregated at the tract level (e.g., counts or densities of specific POI categories) and combined with OD flows to form a unified, feature-rich OD dataset suitable for gravity and Deep Gravity–style models that exploit geographic context.

This dataset is particularly valuable for research on human mobility modeling, OD flow generation, gravity/deep gravity models, and machine learning applications that require realistic OD matrices with rich contextual features for benchmarking and evaluation.

Processed Dataset

This portal provides a processed version of the New York State OD flows enriched with OSM-derived geographic features and a standardized schema. The processed dataset is organized as one folder per county, and each county folder contains four NumPy .npy arrays that encode OD flows, tract-level features, pairwise distances, and adjacency relations between census tracts.

Processed Data: Download county-level .npy folders (ZIP)

The processed dataset is distributed as a ZIP archive. After extraction, each county directory contains four NumPy arrays: od_flow.npy, features.npy, dist.npy, and adj.npy (exact filenames may vary in your implementation).

Data Schema

The processed NYS Mobility Flows dataset is stored in a directory-based layout. For each county, there is a dedicated folder containing four NumPy .npy arrays. All arrays within a county share the same tract ordering along their first (and, where applicable, second) dimension, so that indices are consistent across flows, features, distances, and adjacency.

File Shape Description Example
OD flow matrix (od_flow.npy) (N, N) OD flow counts between census tracts within the county. Entry [i, j] is the number of trips from tract i to tract j over the aggregated time window. e.g., od_flow[3, 7] = 124
Feature matrix (features.npy) (N, F) Tract-level feature vectors. Each row corresponds to a census tract, and each column corresponds to a feature (e.g., population, ACS indicators, and OSM-derived POI or land-use statistics). e.g., features[3] is the feature vector for tract i = 3
Distance matrix (dist.npy) (N, N) Pairwise distances between census tracts in kilometers (e.g., centroid-to-centroid great-circle distance). Entry [i, j] is the distance from tract i to tract j. e.g., dist[3, 7] = 5.3
Adjacency matrix (adj.npy) (N, N) Binary or weighted adjacency between census tracts, representing the underlying graph structure (e.g., shared boundary or network connectivity). Entry [i, j] is non-zero if tracts i and j are adjacent/connected. e.g., adj[3, 7] = 1 if tracts 3 and 7 are neighbors

Conceptually, these four arrays for each county encode the same information that would appear in a tract-level OD table with flow, distance, feature, and adjacency fields, but in a dense matrix form optimized for numerical computation (e.g., PyTorch, NumPy).
