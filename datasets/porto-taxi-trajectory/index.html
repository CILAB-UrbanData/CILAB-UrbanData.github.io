Dataset Description

This dataset contains one year of taxi trajectories collected from 442 taxis operating in Porto, Portugal, between July 1, 2013 and June 30, 2014. Each row corresponds to a completed trip and includes an anonymized trip identifier, taxi identifier, a start timestamp, service type indicators, and a polyline describing the GPS trajectory of the trip.

The trajectory of each trip is encoded as a list of WGS84 longitude–latitude pairs sampled at regular time intervals along the route. Combined with the metadata fields, the dataset can be used to study urban mobility patterns, demand distribution, dispatch strategies, route choice, and congestion, as well as to build machine learning models for destination and travel time prediction.

The version catalogued here is the Taxi Trajectory Data dataset hosted on Alibaba Cloud Tianchi, and it follows the license and terms of use specified on the Tianchi dataset page.

Data Schema

Field Name Data Type Description Example
TRIP_ID string Unique identifier for each completed taxi trip. 1372636858620000589
CALL_TYPE string Service initiation type: 'A' (central dispatch), 'B' (stand), or 'C' (street hail/other). C
ORIGIN_CALL integer / null Anonymized identifier of the caller when the trip was dispatched from the call center (CALL_TYPE = 'A'); null otherwise. 135492
ORIGIN_STAND integer / null Identifier of the taxi stand when the trip started from a stand (CALL_TYPE = 'B'); null otherwise. 7
TAXI_ID integer Identifier for the taxi that performed the trip. 20000589
TIMESTAMP integer Unix timestamp (seconds since 1970-01-01) for the trip start time. 1372636858
DAY_TYPE string Day category of the trip start (e.g., normal day, holiday, or day before a holiday), encoded as 'A', 'B' or 'C'. A
MISSING_DATA boolean Indicates whether the GPS trajectory for this trip has missing segments (TRUE if any points are missing, FALSE otherwise). False
POLYLINE string JSON-like list of GPS points representing the trajectory, where each element is [LONGITUDE, LATITUDE]. One point is recorded at fixed time intervals along the trip. [[ -8.6186, 41.1414 ], [ -8.6185, 41.1414 ], ...]
