Data Schema

Important: traj_*.csv files are headerless and have exactly 4 columns. Interpret fields by column position. Columns 3–4 contain serialized lists (with commas).

Trajectory files: traj_train.csv / traj_valid.csv / traj_test.csv

Column Type Description Example (truncated)
1 string Moving object id (vehicle/taxi identifier). object_001
2 string Trajectory id. traj_12345:object_001
3 string (serialized list) Offsets: serialized list of two tuples (source & destination). Each tuple: (distance_from_segment_start, depart_or_arrive_time, lon, lat). [(12.3, 1477959000, 104.0667, 30.5728), (98.7, 1477959300, 104.0701, 30.5742)]
4 string (serialized list) Segment sequence: serialized list of segment records. Each record: [segment_id, enter_timestamp, average_speed, duration]. The effective segment end time is enter_timestamp + duration. [[123, 1477959000.0, 8.2, 12.0], [456, 1477959012.0, 7.9, 25.0], ...]

Road network files: map/nodes.shp & map/edges.shp

  • nodes.shp: OSM node information (graph nodes).
  • edges.shp: OSM edge/segment information (graph edges). Total edges: 107,655.

Parsing tip: because columns 3–4 contain list strings with commas, use a proper CSV parser (e.g., Python csv/pandas) rather than splitting by comma.
